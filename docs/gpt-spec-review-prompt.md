# GPT仕様レビュー依頼プロンプト

以下をChatGPTに貼り付けて、`docs/spec-review-pack.md` の内容を続けて貼ると、アプリ仕様のレビューを受けやすくなります。

---

あなたは、統計検定1級・統計数理の学習設計、教育工学、復習設計、iPad学習UX、ローカルPWA設計に詳しいレビュー担当者です。

これから、統計検定1級 Study Tracker という学習管理アプリの仕様を渡します。

目的は、アプリが「問題 → 答案作成 → GPT採点 → フィードバック取り込み → 復習 → 弱点分析」のサイクルを、学習効率の面で高く回せる設計になっているかを確認することです。

## 前提

- 主な利用端末はiPadです。
- PCなしで運用できることを重視します。
- OpenAI APIは使いません。
- GPT採点結果は、ユーザーがChatGPTからコピーしてアプリに貼り付けます。
- アプリはPWAで、学習記録は基本的に端末内のIndexedDBに保存します。
- PDF自動解析やOCRは現時点では狙いません。
- 問題マスターと模範解答索引は、ユーザー確認済みのJSONを正本にします。
- まずは、毎日の学習サイクルが破綻なく回ることを最優先にします。

## レビューしてほしいこと

以下の観点で、率直にレビューしてください。

1. 学習効率
   - 統計検定1級の合格に向けて、復習間隔・復習方法・弱点分析は妥当か。
   - K/W/N/Cの分類は実用的か。
   - skeleton / main_calc / full の分け方は、暗記逃げや過剰復習を防げているか。

2. UX
   - iPadだけで運用しやすいか。
   - 今日やること、復習予定、GPT回答取り込み、ダッシュボードの役割分担は分かりやすいか。
   - 表示情報が多すぎる、または少なすぎる箇所はないか。

3. GPT取り込み
   - YAML取り込み仕様は堅いか。
   - problem_id補正、旧キー補完、none判定、next_actionの日付禁止は妥当か。
   - 手動コピペ運用で起きそうな事故は何か。

4. 復習設計
   - ネタバレ防止と復習効率のバランスはよいか。
   - 前回ミス、1行ヒント、保存済みGPT解説、公式解答の参照ルールは妥当か。
   - 「表示を隠してから白紙で再現する」というルールは実用的か。

5. 問題マスター設計
   - PDFを正本にせず、problem_master / answer_index JSONを正本にする設計は安全か。
   - 問題ID不整合やテーマ混入への対策は十分か。

6. 進捗・戦略
   - 残り日数に応じたS/A/過去問の配分は妥当か。
   - 今日の予定時間、先送り候補、ダッシュボードの次の課題表示は学習行動につながるか。

## 出力形式

次の形式で答えてください。

### 1. 総合評価

- 学習効率：100点満点
- UXの分かりやすさ：100点満点
- GPT取り込みの堅牢性：100点満点
- iPad運用の現実性：100点満点
- 実装複雑度の適正さ：100点満点

それぞれ、理由を短く説明してください。

### 2. すぐ直すべき重大問題

毎日の運用に支障が出るものだけを、重要度順に挙げてください。

### 3. 2週間以内に改善するとよい点

学習効率や使いやすさを上げる改善案を挙げてください。

### 4. 後回しでよい点

今は実装しなくてよい機能や、入れると複雑になりすぎる機能を挙げてください。

### 5. 入れない方がよい機能

このアプリの目的から見て、むしろ避けた方がよい機能があれば挙げてください。

### 6. 画面・文言の改善案

ユーザーが迷いやすいラベル、表示文、ボタン名があれば、具体的な修正文を提案してください。

### 7. GPT採点プロンプトへの改善案

初回採点、復習採点、main_calc採点、skeleton採点について、プロンプト上の改善点があれば提案してください。

### 8. 最終的に推奨する運用フロー

このアプリを使う場合の、毎日の理想的な流れを5〜8ステップでまとめてください。

### 9. 確認質問

不明点があれば最後に質問してください。
ただし、質問があっても、分かる範囲で暫定評価は必ず出してください。

## 注意

- OpenAI API連携、OCR、PDF自動解析、クラウドDB化は、現時点では前提にしないでください。
- 実装済みかどうか不明なものは、「実装済みと仮定」と書かず、仕様上の評価として述べてください。
- 仕様に書かれていない機能を勝手にあるものとして扱わないでください。
- 学習効率を上げるためなら、遠慮なく問題点を指摘してください。

# iPad版 運用ガイド

公開URL: https://espsdrphs-eng.github.io/stat-1-study-tracker/

## 1. 初回インストール

1. iPadをインターネットへ接続します。
2. Safariで `https://espsdrphs-eng.github.io/stat-1-study-tracker/` を開きます。
3. Safari上部の共有ボタン（四角から上向き矢印）を押します。
4. メニューを下へ動かし、「ホーム画面に追加」を押します。
5. 名前が「統計一級」になっていることを確認し、「追加」を押します。
6. 以後はホーム画面のアイコンから起動します。

初回表示時にアプリ本体がiPadへ保存されます。インストール後は、通信できない場所でも学習記録、復習予定、GPT回答取り込み、弱点ノート、過去問記録を使用できます。

## 2. 普段の使い方

### 学習前

1. ホーム画面から「統計一級」を開きます。
2. 「今日やること」を確認します。
3. 負荷が4.0を超えている場合は、フル答案を骨格モードへ落とすか翌日へ回します。

### 学習後

1. 「学習記録」を開きます。
2. 問題ID、モード、評価、K/W/N/C、ミス、修正ルールを入力します。
3. 「保存する」を押します。
4. Kの場合は翌日復習と関連S問題の確認が自動追加されます。

### GPT回答を取り込む

1. GPTの回答全文をコピーします。
2. 「GPT回答取り込み」へ貼り付けます。
3. 「内容を解析する」を押します。
4. 問題ID、評価、復習日、S確認、弱点ノートを確認します。
5. 内容が正しい場合だけ「保存する」を押します。

GPTとの通信はこのアプリから行いません。コピー＆ペーストした内容の解析はiPad内で行われます。

## 3. データの保存場所

- 学習記録はiPad内のIndexedDBに保存されます。
- GitHub、OpenAI、その他のクラウドへ学習記録を送信しません。
- iPhoneや別のiPadとは同期しません。
- Safariのプライベートブラウズでは使用しないでください。

次の操作ではデータを失う可能性があります。

- iPadの初期化や故障
- Safariの「履歴とWebサイトデータを消去」
- ホーム画面アプリとサイトデータの削除

## 4. バックアップ

少なくとも週1回、または大量に記録した日にJSONバックアップを作成してください。

1. アプリの「設定」を開きます。
2. 「全データ JSON」を押します。
3. 保存先として「ファイル」を選びます。
4. iCloud Drive内に `統計一級バックアップ` フォルダを作り、保存します。
5. ファイル名の日付を確認します。

CSVは表計算用です。完全復元にはJSONを使用します。

## 5. バックアップから復元

1. 「設定」を開きます。
2. 「JSONバックアップを復元」を押します。
3. 「ファイル」から最新のJSONを選びます。
4. 完了メッセージを確認します。
5. ダッシュボード、復習予定、弱点ノートを確認します。

復元すると、その時点の端末内データがバックアップ内容に置き換わります。

## 6. アプリの更新

公開リポジトリの `main` ブランチへ更新が反映されると、GitHub Actionsが自動的に新しいアプリを公開します。

iPad側では次のように更新します。

1. インターネットへ接続します。
2. ホーム画面アプリを開きます。
3. 一度閉じ、もう一度開きます。

画面や処理プログラムは更新されますが、iPad内の学習記録は維持されます。大きな更新前にはJSONバックアップを作成してください。

## 7. セキュリティ

- GitHub PagesのHTTPSでアプリ本体を配信します。
- Content Security Policyで、アプリからの通信先を同一サイトへ制限しています。
- 外部フォント、広告、アクセス解析、OpenAI APIを使用しません。
- GitHubに公開されるのはプログラムだけです。
- 問題の解答履歴、弱点ノート、GPT貼り付け内容はiPad内に残ります。

GitHub Pagesはアクセス時のIPアドレスをセキュリティ目的で記録します。学習記録の内容は送信されません。

## 8. 困ったとき

### 起動できない

インターネット接続中にSafariで公開URLを一度開き直し、ホーム画面へ再追加します。

### 記録が見つからない

Safari版とホーム画面版で同じ公開URLを使っているか確認します。バックアップがある場合はJSONから復元します。

### 更新されない

インターネット接続を確認し、アプリを完全に閉じて再度開きます。それでも変わらない場合はSafariで公開URLを開き直します。
